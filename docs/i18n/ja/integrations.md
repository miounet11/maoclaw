# 連携と自動化

このページでは、`猫爪 / maoclaw` をクライアント、IDE、自動化システムへ接続する方法を説明します。

## 1. RPC モード

もっとも安定した連携入口は:

```bash
pi --mode rpc
```

stdin/stdout 上の JSON Lines で通信します。用途:

- IDE フロントエンド
- 独自デスクトップシェル
- 自動化コントローラ
- Agent プロキシやオーケストレーション層

プロトコル参照:

- [../../rpc.md](../../rpc.md)

## 2. 軽量な自動化

長寿命クライアントが不要なら、まず次で十分です。

```bash
pi -p "このプロジェクトを要約してください"
pi --mode json -p "構造化された結果を返してください"
```

フル RPC クライアントよりはるかに軽く始められます。

## 3. デスクトップシェル

デスクトップアプリと CLI は、同じ Rust ランタイム、Provider モデル、セッションモデルを共有します。  
デスクトップは別バックエンドではなく、同一基盤の製品シェルです。

## 4. 推奨連携戦略

- 最小工数: `-p` または `--mode json`
- セッションとイベント駆動: `--mode rpc`
- 本格 GUI: RPC 上に独自クライアントを構築
